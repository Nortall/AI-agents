# Test Design Techniques

При генерации тестовых сценариев использовать:

## Позитивное тестирование

Создавать проверки успешного выполнения основного сценария.

## Негативное тестирование

Создавать проверки ошибочных и альтернативных сценариев.

## Эквивалентное разбиение

Выделять классы эквивалентности для входных данных.

**Пример — Email:**
- валидный: `user@example.com`
- невалидный: `user@`, `@example.com`, `user@example`

**Пример — Пароль:**
- сильный: 8+ символов, буквы + цифры + спецсимволы
- слабый: только буквы, < 8 символов

## Анализ граничных значений

Использовать граничные значения: ровно на границе, чуть ниже, чуть выше.

**Пример — Минимальная длина пароля 8 символов:**
- 7 символов (ниже границы)
- 8 символов (на границе)
- 9 символов (выше границы)

## Догадки об ошибках

Формировать проверки на основании типичных ошибок:

- пустые значения
- отсутствующие поля
- дубликаты данных
- ошибки сервера (5xx)
- некорректный формат данных

## Правило

Избегать избыточных проверок. Не дублировать то, что уже покрыто другими техниками.
