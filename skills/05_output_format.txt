# Output Format

Использовать следующий формат.

TC-001 – Название

Приоритет: Высокий

Предусловия:
...

Шаги:

1.
2.
3.

Ожидаемый результат:

1. ...
2. ...
3. ...


## Formatting Rules

- Не объединять несколько шагов в один.
- Не объединять ожидаемые результаты.
- Не использовать маркированные списки внутри одного шага.
- Каждый ожидаемый результат относится только к своему шагу.

## Expected Result Style

Ожидаемый результат должен описывать состояние системы после выполнения каждого шага.

Использовать формулировки:

- отображено;
- открыто;
- создано;
- сохранено;
- отправлено;
- доступно;
- недоступно;
- возвращена ошибка.

Не использовать:

- отображается;
- открывается;
- появляется;
- показывается.

Плохо:

• Открывается страница задания.
• Отображается кнопка «Сохранить».

Хорошо:

• Страница задания открыта.
• Кнопка «Сохранить» отображена.


## One Step — One Result

Каждое действие пользователя должно иметь собственный ожидаемый результат.

Недопустимо:

Шаг 1

Открыть страницу.

Шаг 2

Заполнить форму.

Шаг 3

Нажать кнопку.

Ожидаемый результат

Пользователь зарегистрирован.

Правильно:

Шаги:

1. Открыть страницу.
2. Заполнить форму.
3. Нажать кнопку.

ОР:

1. Страница регистрации открыта.
2. Поля заполнены корректными значениями.
3. Пользователь зарегистрирован.

---

## Language Rules

Использовать деловой технический стиль.

Избегать:

- возможно;
- вероятно;
- скорее всего;
- при необходимости;
- если получится.

Формулировки должны быть однозначными и проверяемыми.
