Test Cases Quality Guard                                                                                                                                                    
                                                                                                                                                                                
    Назначение: анти-фрагментация при генерации тестовых сценариев. Применяется ПЕРЕД выводом результата.                                                                       
                                                                                                                                                                                
    Output Discipline (STRICT)                                                                                                                                                  
                                                                                                                                                                                
    • Выводить ТОЛЬКО итоговый список тест-кейсов                                                                                                                               
    • НЕ выводить черновики или промежуточные наборы                                                                                                                            
    • НЕ перечислять «Исправленный вариант» в конце ответа                                                                                                                      
    • Группировка и удаление дубликатов — ВНУТРЕННЕ (до формирования ответа)                                                                                                    
                                                                                                                                                                                
    Self-Check Algorithm (до вывода)                                                                                                                                            
                                                                                                                                                                                
      Шаг                  Что делать                                                                                                                                           
      ───────────────────  ──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────       
      1. Fragmentation     Посчитать ratio: кол-во тестов : кол-во ключевых требований. Если > 3:1 → сгруппировать. Один UI-элемент в одном контексте = 1–2 теста               
      check                максимум. Бюджетные поля (балл + места + рекомендация + скрытие) → объединить в 2–3 теста, не 8.                                                     
      2. Atomicity         Каждый тест = одна бизнес-идея, а не одна строчка требований. «Пользователь видит корректную карточку» — это бизнес-идея. «Город не выводить         
      re-check             при NULL» — это строчка требований, не бизнес-идея.                                                                                                  
      3. Coverage check    Пройти по каждому критерию приёмки — каждый покрыт хотя бы одним тестом.                                                                             
      4. Output quality    Нет тривиальных «кнопка отображена»; вопросы и предположения маркированы.                                                                            
      check                                                                                                                                                                     
                                                                                                                                                                                
    Target                                                                                                                                                                      
                                                                                                                                                                                
    20–25 тестов на типичный фич-спринт. Если 30+ → fragmentation check.                                                                                                        
                                                                                                                                                                                
    Anti-patterns                                                                                                                                                               
                                                                                                                                                                                
      Антипаттерн             Признак                                       Решение                                                                                             
      ──────────────────────  ────────────────────────────────────────────  ────────────────────────────────────                                                                
      Полевой тест            «Поле X отображено/скрыто»                    Объединить с контекстом                                                                             
      Дублирующий happy path  Один happy path для разных ролей              Объединить в один тест                                                                              
      Контекстный разброс     Один тест на каждый контекст для одного поля  Объединить если поведение одинаковое                                                                
      Zero-test               Тест на value=0 для каждого поля              Объединить все zero/null edge-кейсы                                                                 
      Button-only             «Кнопка отображена» без действия              Объединить с navigation/happy path
