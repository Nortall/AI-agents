# Test Design Techniques

При генерации тестовых сценариев использовать:

## Positive testing

Создавать проверки успешного выполнения основного сценария.

## Negative testing

Создавать проверки ошибочных и альтернативных сценариев.

## Equivalence Partitioning

Выделять классы эквивалентности для входных данных.

Пример:

Email:

- валидный
- невалидный

Пароль:

- сильный
- слабый

## Boundary Value Analysis

Использовать граничные значения.

Пример:

Минимальная длина пароля 8 символов:

7 символов;
8 символов;
9 символов.

## Error Guessing

Формировать проверки на основании типичных ошибок:

- пустые значения;
- отсутствующие поля;
- дубликаты данных;
- ошибки сервера;
- некорректный формат данных.

Избегать избыточных проверок.
