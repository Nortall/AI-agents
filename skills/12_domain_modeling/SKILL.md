# Skill: Domain Modeling

## Цель

Перед генерацией тестовой документации построить внутреннюю модель предметной области.

Задача данного этапа — не генерировать тест-кейсы, а определить структуру системы, выделить ключевые сущности, их свойства, связи и жизненный цикл.

Результаты анализа используются последующими этапами генерации.

---

# Алгоритм

Для каждого требования выполнить следующие шаги.

## Шаг 1. Выделение сущностей

Определить все объекты предметной области.

Примеры:

- Пользователь
- Заказ
- Карточка
- Категория
- Изображение
- Файл
- Проект
- Период
- Новость
- Документ

Каждая существительная, имеющая собственное состояние или поведение, рассматривается как потенциальная сущность.

---

## Шаг 2. Выделение атрибутов

Для каждой сущности определить все описанные характеристики.

Например

Карточка:

- название
- описание
- изображение
- дата
- ссылка
- статус

---

## Шаг 3. Определение операций

Для каждой сущности определить поддерживаемые действия.

Например

Пользователь:

- создание
- редактирование
- удаление
- просмотр

Документ:

- загрузка
- скачивание
- удаление

Изображение:

- загрузка
- замена
- удаление

---

## Шаг 4. Определение состояний

Если требования содержат жизненный цикл объекта, определить его состояния.

Например

Документ:

- создан
- опубликован
- архивирован
- удалён

Если состояния отсутствуют — не придумывать их.

---

## Шаг 5. Построение связей

Определить отношения между сущностями.

Примеры:

Пользователь → создаёт → Проект

Период → содержит → Карточки

Карточка → отображается → Каталог

Категория → включает → Карточки

Изображение → принадлежит → Карточке

---

## Шаг 6. Определение зависимостей

Для каждой сущности определить, какие объекты используют её данные.

Например

Карточка используется:

- каталогом
- поиском
- фильтрацией
- главной страницей

Дата используется:

- сортировкой
- календарём

Изображение используется:

- списком
- детальной страницей

---

# Ограничения

Не придумывать сущности, отсутствующие в требованиях.

Допускается делать только логические выводы, непосредственно следующие из требований.

Запрещено добавлять бизнес-логику, если она явно не описана.

---

# Результат

После завершения анализа должна быть сформирована внутренняя модель системы, содержащая:

- список сущностей;
- атрибуты сущностей;
- операции;
- состояния;
- связи;
- зависимости.

Данная модель используется исключительно как внутреннее представление и не включается в итоговую тестовую документацию.
