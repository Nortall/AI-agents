# Шаблон тестового сценария

## Назначение

Все тестовые сценарии должны формироваться в структуре, совместимой с системой управления тестированием Test IT.

---

# Общий шаблон

Каждый тестовый сценарий должен содержать следующие атрибуты:

- Наименование
- Приоритет
- Предусловия
- Шаги
- Ожидаемый результат для каждого шага
- Постусловия (если требуются)

---

# Правила наименования

Название должно:

- кратко описывать проверяемую функциональность;
- содержать одну бизнес-идею;
- быть понятным без чтения шагов.

Хорошо:

Регистрация пользователя с корректными данными

Плохо:

Проверка регистрации

---

# Приоритет

Каждый тест должен содержать приоритет.

Допустимые значения:

- Высокий
- Средний
- Низкий

Приоритет определяется согласно документу "08_priority_rules".

---

# Предусловия

Предусловия описывают состояние системы до начала выполнения теста.

Правила:

- каждое предусловие оформляется отдельным пунктом;
- предусловия должны быть проверяемыми;
- предусловия не должны содержать действий пользователя.

Хорошо:

1. Пользователь авторизован.
2. В системе существует период обучения.

Плохо:

Пользователь авторизуется.

---

# Шаги

Каждый шаг должен описывать одно действие пользователя.

Правила:

- один шаг — одно действие;
- действия должны выполняться последовательно;
- шаг должен быть воспроизводимым;
- не объединять несколько действий в одном шаге.

Хорошо:

1. Открыть страницу регистрации.

2. Заполнить поле Email.

3. Нажать кнопку "Создать".

Плохо:

Открыть страницу, заполнить форму и нажать кнопку.

---

# Ожидаемый результат

Количество пунктов верхнего уровня в секции «Ожидаемый результат» должно **строго соответствовать** количеству шагов в секции «Шаги».

**Жёсткое правило 1:1:** см. `03_quality_and_best_practices` (раздел «Пошаговая проверка») — единый источник истины для всех правил 1:1.

**Структура:**
Каждый шаг (1, 2, 3...) имеет свой пункт ожидаемого результата (1, 2, 3...).

**Пример:**
Шаги:
1. Кликнуть на поисковую строку.
2. Ввести текст.
3. Нажать Enter.

Ожидаемый результат:
1. Отображён выпадающий список подсказок.
   * Подсказки отображены для введённых символов.
   * Максимальное число — 5.
   * Совпадения выделены.
2. Страница результатов открыта.
   * Подразделы отображены в списке.
   * Результаты отображены в блоках.
3. Значение сохранено в историю.
   * Запись отображена в «Search history».

**Правила:**
- Шаг 1 → Ожидаемый результат 1. Шаг 2 → Ожидаемый результат 2.
- Никогда не создавайте больше пунктов ожидаемого результата, чем шагов.
- Если шаг содержит несколько аспектов проверки — они объединяются вложенными буллитами под номером ожидаемого результата.

---

# Стиль ожидаемого результата

Ожидаемый результат описывает состояние системы после выполнения конкретного шага.

Использовать формулировки прошедшего времени (пасивные конструкции):
- отображено;
- открыто;
- создано;
- сохранено;
- возвращена ошибка;
- сообщение отображено;
- запись создана;
- данные обновлены;
- заблокировано.

Не использовать:
- отображается;
- открывается;
- появляется;
- показывается;
- выводится;
- становится доступным.

Плохо:
• Открывается страница задания.
• Отображается кнопка «Сохранить».
• Подсказки отображаются под полем.

Хорошо:
• Страница задания открыта.
• Кнопка «Сохранить» отображена.
• Подсказки отображены под полем.

---

# Постусловия

При необходимости указывать итоговое состояние системы после выполнения теста.

Пример:

Созданный пользователь удален из базы данных.

---

# Группировка проверки валидации (Анти-фрагментация)

Валидация одного поля (пустое значение, минимальная длина, формат, спецсимволы) = **ОДИН тест-кейс**.

Недопустимо:
- Разбивать валидацию одного поля на два и более теста (например: пустое → TC-006, минимальная длина → TC-007).
- Использовать секцию «Дополнительные шаги» — все шаги идут подряд в одном блоке «Шаги».
- Один шаг = одно действие, но все варианты валидации одного поля объединяются в один тест.

Правильно:
```
Шаги:
1. Оставить поле пустым.
2. Нажать кнопку «Сохранить».
3. Ввести значение длиной менее X символов.
4. Нажать кнопку «Сохранить».

Ожидаемый результат:
1. Отображено сообщение об ошибке: поле обязательно.
2. Изменения не сохранены.
3. Отображено сообщение об ошибке: минимальная длина не соблюдена.
4. Изменения не сохранены.
```

**Ожидаемый результат** всегда пронумерован 1-N и строго привязан к номеру шага (Ожидаемый результат 1 → Шаг 1, Ожидаемый результат 2 → Шаг 2 и т.д.).
