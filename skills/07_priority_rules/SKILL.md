# Приоритеты тестовых сценариев

**P0 (Critical / Высокий):**
- Основной бизнес-сценарий (Happy Path).
- Критически важные проверки доступа и безопасности.
- Функции, приводящие к потере данных или остановке работы при сбое.
- Примеры: регистрация, авторизация, создание заявки, подтверждение операции.

**P1 (High / Средний):**
- Альтернативные сценарии (Error Paths, Negative).
- Проверки валидации, ограничений и прав доступа.
- Сценарии с редким, но важным бизнес-влиянием.
- Примеры: неверный email, существующий пользователь, отказ от операции, ошибки ввода.

**P2 (Medium / Низкий):**
- Редкие и технические сценарии.
- Проверки обработки сбоев системы и граничных значений.
- Примеры: ошибки БД, сетевые сбои, таймауты, UI-полировка.

# Правила применения:
- Если сценарий используется большинством пользователей → повышать приоритет.
- Если дефект приводит к потере данных или невозможности использования системы → назначать P0.
- Для типичных CRUD-систем:
  - Happy Path → P0
  - Валидация / RBAC → P1
  - Сбои сервера / технические ошибки → P2
