# Формат вывода

Использовать следующий формат.

TC-001 – Название

Приоритет: Высокий

Предусловия:
...

Шаги:

1.
2.
3.

Ожидаемый результат:

1. ...
   * детализация 1
   * детализация 2
2. ...
   * детализация 1
3. ...

---

## Правила форматирования

- Не объединять несколько шагов в один.
- Количество пунктов верхнего уровня в «Ожидаемый результат» должно **строго соответствовать** количеству шагов.
- Если шаг содержит несколько аспектов проверки — они описываются вложенными буллитами под соответствующим пунктом ожидаемого результата.
- Каждый ожидаемый результат относится только к своему шагу.

---

## Ожидаемый результат

Количество пунктов верхнего уровня в секции «Ожидаемый результат» должно **строго соответствовать** количеству шагов в секции «Шаги».

**Структура:**
Каждый шаг (1, 2, 3...) имеет свой пункт ожидаемого результата (1, 2, 3...).

**Пример:**
Шаги:
1. Кликнуть на поисковую строку.
2. Ввести текст.
3. Нажать Enter.

Ожидаемый результат:
1. Отображён выпадающий список подсказок.
   * Подсказки отображены для введённых символов.
   * Максимальное число — 5.
   * Совпадения выделены.
2. Страница результатов открыта.
   * Подразделы отображены в списке.
   * Результаты отображены в блоках.
3. Значение сохранено в историю.
   * Запись отображена в «Search history».

**Правила:**
- Шаг 1 → Ожидаемый результат 1. Шаг 2 → Ожидаемый результат 2.
- Никогда не создавайте больше пунктов ожидаемого результата, чем шагов.
- Если шаг содержит несколько аспектов проверки — они объединяются вложенными буллитами под номером ожидаемого результата.

---

## Стиль ожидаемого результата

Ожидаемый результат должен описывать состояние системы после выполнения каждого шага.

Использовать формулировки прошедшего времени (пассивные конструкции):
- отображено;
- открыто;
- создано;
- сохранено;
- отправлено;
- доступно;
- недоступно;
- возвращена ошибка;
- сообщение отображено;
- запись создана;
- данные обновлены;
- заблокировано.

Не использовать:
- отображается;
- открывается;
- появляется;
- показывается;
- выводится;
- становится доступным;
- становится видимым.

Плохо:
• Открывается страница задания.
• Отображается кнопка «Сохранить».
• Подсказки отображаются под полем.

Хорошо:
• Страница задания открыта.
• Кнопка «Сохранить» отображена.
• Подсказки отображены под полем.

---

## Один шаг — один результат

Каждое действие пользователя должно иметь собственный ожидаемый результат.

Недопустимо:

Шаг 1
Открыть страницу.

Шаг 2
Заполнить форму.

Шаг 3
Нажать кнопку.

Ожидаемый результат
Пользователь зарегистрирован.

Правильно:

Шаги:
1. Открыть страницу.
2. Заполнить форму.
3. Нажать кнопку.

ОР:
1. Страница регистрации открыта.
2. Поля заполнены корректными значениями.
3. Пользователь зарегистрирован.

---

## Языковые правила

Использовать деловой технический стиль.

Избегать:
- возможно;
- вероятно;
- скорее всего;
- при необходимости;
- если получится.

Формулировки должны быть однозначными и проверяемыми.
