# Критичность бизнес-функции

# Высокий:

Основной бизнес-сценарий.

Примеры:

регистрация;
авторизация;
оформление заказа;
оплата.

# Средний:

Альтернативные сценарии и проверки валидации.

Примеры:

неверный email;
существующий пользователь;
слабый пароль.

# Низкий:

Редкие и технические сценарии.

Примеры:

ошибки БД;
сетевые сбои;
таймауты.


# Частота использования:

Если сценарий используется большинством пользователей:

повышать приоритет.

Потенциальный ущерб

Если дефект приводит к:

потере данных;
невозможности использования системы;
финансовым потерям;

назначать High.

# Для типичных CRUD-систем:

Happy path → Высокий

Validation → Средний

Server failures → Низкий
